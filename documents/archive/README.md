# Documentos de Kick-off (Archive)

Esta pasta contém os **documentos originais** usados no kick-off do projeto.

---

## Propósito

Estes documentos servem como:
1. **Fonte de contexto** para preenchimento automático dos templates
2. **Registro histórico** das decisões iniciais
3. **Referência** quando houver dúvidas sobre requisitos originais

---

## O Que Colocar Aqui

### Documentos Essenciais

| Documento | Formato | Descrição |
|-----------|---------|-----------|
| **TAP** | PDF, Word, TXT | Termo de Abertura do Projeto |
| **Transcrições** | TXT | Reuniões de kick-off, discovery, alinhamentos |
| **Especificações** | PDF, MD | Requisitos técnicos, funcionais |

### Documentos Opcionais

| Documento | Formato | Descrição |
|-----------|---------|-----------|
| Mockups/Wireframes | PNG, PDF | Esboços de interface |
| Diagramas | PNG, PDF | Arquitetura proposta |
| Análises | PDF, XLSX | Dados de mercado, concorrência |
| Contratos | PDF | Termos de serviço, SLAs |

---

## Formato Recomendado

### Para Transcrições

Nomeie os arquivos com data e contexto:
```
2026-01-20_kickoff_reuniao.txt
2026-01-22_discovery_stakeholders.txt
2026-01-25_alinhamento_tecnico.txt
```

### Para PDFs/Documentos

Use nomes descritivos:
```
TAP_NomeProjeto_v1.pdf
Especificacao_Tecnica_v1.pdf
Requisitos_Funcionais_v1.pdf
```

---

## Importante

> **Estes documentos são históricos.**
> Para decisões atuais, consulte sempre `documents/core/Projeto.md`.

Se houver conflito entre documentos de archive e Projeto.md:
- **Projeto.md prevalece** (é o living document)
- Archive representa o estado inicial
- Mudanças devem ser documentadas no changelog de Projeto.md

---

## Após o Kick-off

Depois que os templates forem preenchidos:

1. **Mantenha os arquivos aqui** - São referência histórica
2. **Não edite** - São snapshot do início do projeto
3. **Consulte Projeto.md** - Para informações atualizadas

---

## Confidencialidade

Alguns documentos podem conter informações sensíveis:
- Valores financeiros
- Dados de clientes
- Estratégias de negócio

Certifique-se de que o repositório tem as permissões adequadas.
